# Plan
## 总览
目标：三个月后完成一个完整的游戏服务器 Demo，具备面试能力。  
路线：网络 IO → 并发模型 → 协议 → Redis → MySQL → 架构 → Demo → 压测与面试准备

---

## 第 1 个月：网络 IO + 并发（核心补齐）

### 第 1 周：网络 IO
- 学习 epoll（LT/ET 模式）
- 实现非阻塞 accept/read/write
- 粘包/拆包处理
- 心跳检测

**产出**：一个 epoll 服务器，支持多连接与心跳。

---

### 第 2 周：多线程与并发模型
- 实现线程池（固定线程数 + 任务队列）
- 学习 CAS、atomic、false sharing
- epoll + 线程池结合，消息分发

**产出**：多线程 epoll 服务器。

---

### 第 3 周：协议设计
- 自定义二进制协议（包头：长度 + cmd）
- 消息路由（cmd → handler）
- 可选：Protobuf 序列化

**产出**：协议解析器 + 消息分发器。

---

### 第 4 周：服务器基础功能
- 连接管理（玩家上下线）
- 定时器（心跳、超时）
- 日志系统（spdlog）
- 配置系统（json/yaml）

**产出**：一个可扩展的服务器框架。

---

## 第 2 个月：Redis + MySQL + 架构

### 第 5 周：Redis
- 掌握 string/hash/list/set/zset
- 排行榜（zset）
- 匹配队列（list）
- 在线状态管理（hash）
- Lua 脚本实现原子操作

**产出**：Redis 模块（排行榜、匹配、在线状态）。

---

### 第 6 周：MySQL
- CRUD、索引、事务
- 玩家数据表设计（玩家、背包）
- 实现连接池

**产出**：玩家数据加载/保存模块。

---

### 第 7 周：游戏服务器架构
- 网关（Gateway）
- 逻辑服（Logic）
- 匹配服（Match）
- 房间服（Room）

**产出**：架构图 + 网关与逻辑服通信。

---

### 第 8 周：功能实现
- 登录
- 心跳
- 聊天
- 玩家数据加载/保存
- 匹配 → 房间创建 → 广播

**产出**：完整的功能链路。

---

## 第 3 个月：Demo + 压测 + 面试准备

### 第 9–10 周：基础功能完善
- GM 命令（加金币）
- 在线状态（Redis）
- 玩家数据持久化

**产出**：基础服务器 Demo。

---

### 第 11 周：匹配 + 房间系统
- 匹配队列（Redis）
- 房间创建（Redis hash）
- 房间广播
- 玩家加入/退出房间

**产出**：匹配成功 → 创建房间 → 通知玩家进入。

---

### 第 12 周：战斗同步 + 压测 + 面试准备
- 简单帧同步（可选）
- 压测（1000+ 并发连接）
- 完善日志/配置
- 面试题准备（Redis、协议、架构、并发）

**产出**：完整的游戏服务器 Demo + GitHub 项目 + 简历项目描述。

---

## 最终效果
三个月后你将具备：
- 写高并发 epoll 服务器
- 写线程池
- 设计协议
- 使用 Redis 做匹配/排行榜/在线状态
- 使用 MySQL 做玩家数据
- 写一个完整的游戏服务器 Demo
- 面试时能自信回答架构、并发、协议、Redis/MySQL相关问题
